# User Persona Generator

**Category:** ux-research
**Difficulty:** Intermediate
**Tags:** #ux-research #personas #user-research #e-commerce #beauty #bilingual

## Description

Generate detailed, data-driven user personas for beauty e-commerce platforms. Includes skin concern profiles, device preferences, language behavior (English/Arabic), shopping patterns, and how each persona interacts with the Digital Concierge "3-Click Solution."

## Prompt

Create detailed user personas for my [project type] with the following context:

PROJECT CONTEXT:
- Platform: [beauty e-commerce, wellness app, pharma retail, etc.]
- Market: [Jordan, GCC, MENA, global]
- Languages: [English, Arabic, or bilingual]
- Product range: [5,000+ SKUs: dermo-cosmetics, makeup, supplements]
- Key feature: [AI Digital Concierge with 3-Click Skin Solution]

PERSONA REQUIREMENTS:

For each persona, provide:

1. DEMOGRAPHIC PROFILE:
   - Name, age, location
   - Occupation and income level
   - Primary language and device
   - Tech-savviness level

2. BEAUTY PROFILE:
   - Primary skin concern (Acne, Anti-Aging, Dryness, Pigmentation)
   - Current skincare routine maturity (basic, moderate, advanced)
   - Preferred brands (dermo-cosmetics vs. mass market)
   - Budget range per transaction

3. SHOPPING BEHAVIOR:
   - Primary discovery channel (Instagram, WhatsApp referral, Google)
   - Decision-making style (research-heavy vs. impulse)
   - Trust signals that matter most (authenticity guarantee, pharmacist approval, reviews)
   - Preferred payment method (COD vs. card)

4. DIGITAL CONCIERGE INTERACTION:
   - Likely entry question to the AI
   - Expected path through the "3-Click Solution"
   - Potential drop-off point
   - Preferred language for AI chat

5. FRICTION POINTS & SOLUTIONS:
   - Where they abandon the flow
   - What "Authentic Quality" messaging resonates
   - How the pharmacist-curated framing builds trust

OUTPUT FORMAT:
- Full narrative persona card
- Behavior summary table
- Journey map for the "3-Click Solution"
- Design implications for the UI

## Example Usage

**Input:**
```
Platform: Asper Beauty Shop (Jordan-based, ships GCC)
Languages: English primary, Arabic secondary
Key personas needed:
  1. Young Jordanian woman, acne-prone, Instagram-driven
  2. Expat professional, anti-aging focus, research-heavy
  3. Arabic-speaking mother, sensitive skin + baby products
```

## Sample Results

**Persona 1 — "Sara, The Informed Millennial"**
- Age 26, Amman, bilingual (Arabic-first), uses Instagram daily
- Skin concern: Acne + oiliness, uses La Roche-Posay Effaclar
- Finds Asper via Instagram reel, clicks "Ask AI" button on product
- 3-Click path: Acne → Oily skin → Effaclar Duo+ recommendation
- Drops off if: checkout requires registration before COD option
- Trust signal: "Pharmacist-Curated" badge + authentic sourcing guarantee
- Design implication: Prominent COD option, social proof on product cards

**Persona 2 — "Nadia, The Research-First Professional"**
- Age 38, Dubai expat, English-primary, uses desktop
- Skin concern: Anti-aging, prefers Vichy LiftActiv range
- Googles "authentic Vichy Jordan" — finds Asper
- 3-Click path: Anti-Aging → Dry skin → LiftActiv Vitamin C Serum
- Drops off if: no ingredient transparency or missing INCI list
- Trust signal: Clinical ingredient explanations, authentic sourcing
- Design implication: "Ingredients" expandable section on product page

## Notes

- Jordanian market has high COD preference (70%+) — reflect in personas
- Arabic personas prefer Tajawal font and RTL layout — note UI implications
- The "Pharmacist Authority" framing works strongly for medical-adjacent concerns
- Include both English and Arabic persona variants where relevant
- Personas should drive specific UI decisions (card layout, quiz wording, CTA copy)
